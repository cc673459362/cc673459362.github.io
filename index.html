windows_cpu_state

Posted on 2021-01-27

windows cpu state

1. ready: thread get the all resources except cpu.

2. running: complete context exchange, and into running state until thread die or be take over by other tasks.

3. wating: thread waiting for something happened.

4. standby: thread choose one cpu core and wait for context exchange.

5. transition: thread is waiting for running , but some stack or heap is out of memory.

6. terminated: thread terminate.

7. initialized: thread init.

Windows调起外部EXE 的Shellexecute踩坑

Posted on 2020-10-27

1. 在被调进程中用Windows API 获取当前目录获取到的是调起进程目录。

必须要用getmodulefilename 这个API可以获取到自己这个exe所在的目录。

2. WINDOWS API中的字符串类型LOCWSTR转换问题。

Windows有些系统API的参数是WINDOWS宏定义出来的类型。
在使用ShellExecute调起外部EXE进程时，文件目录参数
typedef struct _SHELLEXECUTEINFOW {
DWORD cbSize;
ULONG fMask;
HWND hwnd;
LPCWSTR lpVerb;

LPCWSTR lpParameters;
LPCWSTR lpDirectory;
int nShow;

void *lpIDList;

HKEY hkeyClass;
DWORD dwHotKey;
union {
HANDLE hIcon;
HANDLE hMonitor;
} DUMMYUNIONNAME;

} SHELLEXECUTEINFOW, *LPSHELLEXECUTEINFOW;
